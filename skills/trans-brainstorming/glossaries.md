# Domain Glossary Templates

Use this file to bootstrap terminology agreements during the clarifying-questions phase.

When a domain is identified, pull the relevant section and confirm key term mappings with the user BEFORE translating.

---

## How to Use

1. Identify the source domain.
2. Pull the relevant section below.
3. Ask: "Here are the key terms I'll be working with. Do any of these have preferred translations in your context?"
4. Record agreed mappings in the Translation Brief.

## Adding a New Domain

When a project uses a domain not listed here, build a mini-glossary during the clarifying phase:

1. Extract 8-12 key terms from the source.
2. Propose target-language translations.
3. Ask the user to confirm or override each.
4. Record final mappings in the Translation Brief.
5. Optionally append to this file for future reuse.
