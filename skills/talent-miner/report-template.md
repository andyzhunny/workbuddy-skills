# Report Template — 个人天赋使用说明书

## 总体要求

- **篇幅**：10,000字以上。宁可长不可短。每一段都要有信息密度，不注水。
- **风格**：温暖而犀利。像一位足够了解你、也足够诚实的朋友在跟你说话。
- **结构**：不设固定模板，根据用户的实际回答自由组织。以下为参考框架，但**必须根据用户独特的故事来调整结构和重点**。
- **目标**：让用户读完之后觉得"这个人比我自己还了解我"，并且产生"我知道下一步该怎么走了"的感觉。

## 参考结构（根据用户情况自由调整）

### 一、写在前面：为什么要有一份天赋说明书（约 500 字）

不是鸡汤。从一个真实的矛盾切入——为什么一个有能力的人会感到迷茫。天赋被遮蔽的三个原因：

1. **社会规训**：16岁之后，我们学会的不是"我是谁"，而是"世界要我成为谁"。
2. **能力与天赋的混淆**：擅长 ≠ 热爱。你被夸得多的事情，不一定是你的天赋。
3. **阴影恐惧**：我们被教导要修缺点，但缺点往往是天赋的背面。

引出这份说明书的定位：这不是一个测试结果，而是一面用对话慢慢磨出来的镜子。

### 二、你的天赋原石（约 2000-3000 字）

这是整份报告最核心的部分。**根据对话中的实际信号，自由组织，不要套模板。** 以下为写作方向：

**每个天赋点需要覆盖：**

1. **定义**：这不是什么（先破除误解），而是什么（给出精准的定义）
2. **证据链**：从对话中提取 2-3 个具体证据（用户的原话或故事），用它们串联出这个天赋的存在
3. **运行机制**：这个天赋在日常生活中是怎么运作的？当它被激活时，用户会表现出什么行为、产生什么感受？
4. **阴影面**：这个天赋被压抑或过度使用时，会变成什么"缺点"？这个"缺点"曾经给他带来什么困扰？
5. **回血测试**：用能量审计法验证——做这件事时是回血还是耗电？如何区分？

**写作要求：**
- 使用用户在对话中说过的**原话**（引用），让他感到"被看见"
- 每个"缺点"都要翻面，展示它作为天赋时的样子
- 如果用户的证据链指向一个**unexpected**的天赋（比如"爱发呆" → 深度思考能力），要特别花篇幅帮用户接受这个认知转换

### 三、你的天赋操作系统（约 1500-2000 字）

将发现的天赋放在一个**整体框架**中，让用户看到它们如何相互配合。

建议使用"操作系统"隐喻：
- **CPU**：你最核心的天赋（回血最强的那一个）
- **RAM**：你的快速处理能力（无意识胜任区）
- **GPU**：你的并行/直觉/创造力能力
- **存储**：你积累的独特经验和知识结构

或者使用其他更合适的隐喻。关键是让用户**看到一个系统**，而不是一堆零散的优点。

**重点**：分析天赋之间的协同效应和潜在冲突。哪些天赋一起使用时会 1+1>2？哪些会互相内耗？

### 四、你的人生路标（约 2000-3000 字）

这是最"有用"的部分。将天赋翻译成**具体的职业方向和生活建议**。

**职业方向：**
- 列出 3-5 个最匹配天赋的职业方向/赛道
- 对每个方向，分析：为什么匹配（天赋对应关系）、可能的挑战、适合什么环境/阶段
- 特别标注：哪些是"理想选择"，哪些是"现实过渡"，帮用户理解路径

**环境适配：**
- 什么样的工作环境会让你的天赋发光？
- 什么样的环境会压制你的天赋？
- 你需要什么样的团队伙伴来补你的短板？

**风险提示：**
- 你可能犯的最大的"天赋陷阱"是什么？（比如天赋过度使用导致的问题）
- 什么样的职业看似匹配其实有毒？

### 五、一个送给你的问题（约 300-500 字）

收尾不是总结，是一个**问题**。

这个问题是整份报告的最后一锤——把用户推向他需要自己面对的那个路口。不是给答案，是给他一个未来一周、一个月、甚至一年都值得反复想的问题。

可以参考的方向：
- "如果五年后的你回头看今天的自己，你会感谢他做了什么选择？"
- "你现在的生活里，哪件事最接近你16岁时忘我做的那件事？"
- "你的嫉妒在告诉你什么？你打算听吗？"

### 六、附录：信号清单（约 500-1000 字）

将对话中收集到的所有信号整理成一个清单：
- 每个问题的核心发现
- 标注信号类型（回血/无意识胜任/阴影）
- 标注信号强度（强/中/弱）

## 写作禁忌

1. **不套话**：不要出现"你是独一无二的"、"每个人都有自己的光芒"这种正确的废话
2. **不讨好**：如果用户的天赋指向一个他不情愿的方向，诚实说出来
3. **不预测**：不要说"你一定能成功"或"你适合当CEO"——说"你的天赋结构适合___类型的环境/角色"，让用户自己判断
4. **不制造焦虑**：不暗示"如果你不使用天赋就浪费了"。天赋是礼物，不是债务
5. **不忽视阴影**：用户在对话中回避的问题，在报告中要温和地提出来，而不是假装没看到

## 文件输出

生成两份文件：
1. `~/Documents/notes/{timestamp}--天赋说明书__talent-manual.md` （Markdown）
2. `~/Documents/notes/{timestamp}--天赋说明书__talent-manual.org` （org-mode）

org-mode 文件头部：
```org
#+title: 个人天赋使用说明书：{用户昵称}
#+date: [{日期}]
#+filetags: :talent:career:
#+identifier: {timestamp}
```

向用户报告两个文件的路径。
