# 本科商务英语毕业论文批阅

本科商务英语专业毕业论文的系统性批阅工具。自动读取工作空间中的论文文件，按四个维度检查，输出「问题清单 + 修改建议」格式的批阅报告。

## 触发条件

当用户说「论文检查」「论文批阅」「毕业论文检查」「thesis review」或类似表述时，加载此 Skill。

## 使用说明

1. 用户将学生论文所在文件夹设为 WorkBuddy 工作空间
2. 触发 Skill 后，自动扫描工作空间中的论文文件（`.docx`、`.pdf`、`.txt`）
3. 如有多篇论文，逐篇处理，每篇生成独立报告
4. 报告保存为 Markdown 文件到工作空间根目录，命名格式：`批阅报告_[原文件名].md`

## 工作流程

```
扫描工作空间 → 找到论文文件 → 读取全文
→ 维度①论文结构 → 维度②语言质量 → 维度③商务英语专业性 → 维度④论点与论证
→ 生成批阅报告 → 保存到工作空间
```

**文件读取规则**：
- 优先处理 `.docx` 文件，其次 `.pdf`，最后 `.txt`
- 如果工作空间无论文文件，提示用户将论文文件放入工作空间
- 使用 `read_file` 工具读取 `.txt` 文件；对于 `.docx` 和 `.pdf`，使用 `execute_command` 调用相应工具解析（如 `npx mammoth` 读取 docx，`npx pdf-parse` 读取 pdf）

## 检查维度与标准

严格按以下四个维度逐一检查论文。每个维度列出所有发现的问题，未发现问题的维度注明"该维度未发现明显问题"。

### 维度一：论文结构

检查论文是否具备完整、逻辑连贯的学术结构。

**检查清单**：

1. **结构完整性**：是否包含完整的论文结构链：
   - Abstract（摘要）
   - Introduction（引言）
   - Literature Review（文献综述）
   - Methodology（研究方法）
   - Results / Analysis（结果/分析）
   - Discussion（讨论）
   - Conclusion（结论）
   - References（参考文献）
   - 任一必要章节缺失即记录为问题

2. **逻辑连贯性**：各章节之间是否有清晰的逻辑衔接，是否存在明显的逻辑跳跃或内容脱节

3. **Abstract 质量**：
   - 是否概括了研究目的（research purpose）
   - 是否说明了研究方法（methodology）
   - 是否呈现了主要发现（key findings）
   - 是否超过 300 词（本科论文摘要通常 200-300 词）

4. **Introduction 质量**：
   - 是否清晰陈述了研究问题（research question(s)）
   - 是否交代了研究背景和意义
   - 是否概述了论文的整体结构

5. **Conclusion 质量**：
   - 是否回应了引言中提出的研究问题
   - 是否总结研究的主要发现和贡献
   - 是否指出研究局限性和未来研究方向
   - 是否避免了简单重复前文内容

### 维度二：语言质量

检查学术英语写作的语言规范和表达水平。

**检查清单**：

1. **语法正确性**：重点检查以下高频问题：
   - 主谓一致（subject-verb agreement）
   - 时态使用（tense consistency，尤其文献综述中的过去时 vs 一般现在时）
   - 冠词使用（article usage，a/an/the 的误用或缺失）
   - 介词搭配（preposition collocation）
   - 代词指代不清（ambiguous pronoun reference）

2. **词汇选择**：
   - 是否使用口语化/非正式表达（如 "a lot of" → 应为 "a significant number of / numerous"；"thing" → 应为 "aspect / factor / element"）
   - 是否存在中式英语（Chinglish）表达
   - 同一段落内是否过度重复使用同一词汇

3. **句式多样性**：
   - 是否存在过多简单句堆叠（缺乏复合句/复杂句）
   - 是否存在过度冗长复杂的长难句（影响可读性）
   - 句式长度是否有合理的交替变化

4. **学术语体**：
   - 是否保持了客观、正式的学术语气
   - 是否避免使用第一人称口语表达（如 "I think" → "This study argues"；"you can see" → "It is evident that"）
   - 是否使用了模糊限定词（hedging language）来体现学术谨慎性（如 "suggests" rather than "proves"）

5. **衔接手段**：
   - 段落之间是否有恰当的过渡
   - 句子之间是否使用了逻辑连接词（however, furthermore, in contrast, consequently 等）
   - 每段是否有清晰的主题句（topic sentence）

### 维度三：商务英语专业性

检查论文在商务英语领域的专业深度和术语准确性。

**检查清单**：

1. **商务术语准确性**：
   - 专业术语使用是否准确（区分近义术语，如 "stakeholder" vs "shareholder"、"marketing mix" vs "4Ps"、"joint venture" vs "wholly owned subsidiary"）
   - 术语使用是否前后一致（同一概念全文应使用统一术语）
   - 是否存在术语误用或随意使用的情况

2. **研究问题聚焦度**：
   - 研究问题是否明确聚焦于商务英语相关领域，包括但不限于：
     - 国际贸易与商务沟通
     - 跨文化交际与商务礼仪
     - 市场营销与品牌策略
     - 商务翻译与本地化
     - 商务话语分析与体裁分析
     - 企业文化与组织沟通
   - 研究范围是否过大或过泛，缺乏具体聚焦

3. **理论框架**：
   - 是否引用了商务英语或应用语言学相关理论（如 Halliday 的系统功能语言学/语域理论、Swales 的体裁分析/Move 结构、Hofstede 的文化维度理论、Porter 的五力模型等）
   - 理论框架是否与研究问题匹配
   - 理论引用是否为经典或权威来源

4. **数据/案例真实性**：
   - 分析是否基于真实的商务案例、企业数据或语料库
   - 是否存在泛泛而谈、缺乏具体案例支撑的情况
   - 数据来源是否可靠、可追溯

### 维度四：论点与论证

检查论证链的完整性、逻辑性和说服力。

**检查清单**：

1. **核心论点明确性**：
   - 论文是否有明确的 thesis statement（中心论点）
   - thesis statement 是否在 Introduction 中清晰呈现
   - 全文是否围绕 thesis statement 展开

2. **论据充分性**：
   - 每个主要观点是否有充分的支持证据（数据、案例、文献引用）
   - 是否存在"assertion without evidence"（只有论断没有论据）的情况
   - 引用的文献是否具有权威性和时效性

3. **反面观点与局限性**：
   - 是否考虑了与自己论点不同的观点
   - 是否承认了研究的局限性（methodological limitations, scope constraints 等）
   - 对反面观点的处理是否客观公正

4. **结论推导**：
   - 结论是否基于前文论证自然推导得出
   - 是否在结论中引入了前文未讨论的新论点或新证据
   - 结论与引言中的研究问题是否形成闭环

5. **论证链完整性**：
   - 从研究问题 → 文献综述 → 方法 → 分析 → 结论的整体逻辑链是否完整
   - 是否存在逻辑断裂或论证跳跃
   - 各章节之间的论证是否有递进关系

## 输出规则

1. **真实性优先**：只列出论文中实际存在的问题，不要凭推测编造问题。引用原文时必须准确，不得篡改或编造论文内容。
2. **具体化**：每条问题都要标注具体位置（章节名、段落号或页码），引用相关原文，给出可操作的修改建议。
3. **建设性**：修改建议要具体可行，最好包含正确写法的示例，而非泛泛的"需要改进"。
4. **不过度**：不要对同一类问题反复列举。如果某一问题在全文中普遍存在（如时态混乱），汇总为一条，标注出现频率和典型示例即可。
5. **客观中立**：批阅报告是辅助工具，不替代教师的最终判断。避免使用过度严厉或主观的措辞。
6. **文件保存**：报告生成后，使用 `write_to_file` 保存到工作空间根目录。
